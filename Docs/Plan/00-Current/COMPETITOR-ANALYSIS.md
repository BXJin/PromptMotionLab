# Competitor Analysis - AI Character / Digital Human Systems

Updated: 2026-05-14

## Summary

The market already has many products that combine conversational AI with digital humans, avatars, lip-sync, facial expressions, and kiosk/web deployment.

PromptMotionLab should not claim that "no one is doing AI characters." That is false.

The safer and stronger position is:

```text
Existing products are strong in video digital humans, web avatars, and fast SDK deployment.
PromptMotionLab focuses on Behavior JSON-driven, Unreal-first, spatial 3D character runtime for kiosk/exhibition/showroom use cases.
```

## Competitive Categories

| Category | Strength | PromptMotionLab Response |
|---|---|---|
| Video digital humans | realistic face, fast deployment, web/mobile SDK | do not compete only on face realism |
| Browser/VRM 3D avatars | lightweight, cross-platform, low latency | focus on Unreal spatial quality and object interaction |
| Game NPC AI platforms | LLM NPC dialogue, game engine integration | focus on B2B kiosk/exhibition operation and Behavior JSON policy |
| Scripted avatar video tools | cheap video generation, scale | focus on real-time interaction and spatial character control |
| Digital human SDKs | enterprise integration and references | differentiate with renderer-agnostic Behavior JSON and Unreal runtime execution |

Use this positioning:

```text

It focuses on a controllable behavior layer that translates LLM response state into avatar behavior, starting with an Unreal-based spatial 3D character runtime.
```

## Strategic Differentiators

### 1. Behavior JSON as Product Core

Most avatar systems expose:

```text
LLM + TTS + Avatar Renderer
```

PromptMotionLab should expose:

```text
LLM + Behavior Policy Layer + Renderer Adapter
```

Behavior JSON contains:

- intent
- emotional tone
- confidence
- gaze target
- gesture key
- speech style
- safety state

### 2. Multi-Renderer Potential

The first renderer is Unreal, but the Behavior JSON can later map to:

- video avatar APIs
- MetaHuman
- Web/VRM avatars
- Live2D
- browser/on-device avatars

This prevents the product from being locked to one renderer.

### 3. Unreal Spatial Runtime

Unreal is valuable when the character must exist inside a space:

- exhibition guide
- showroom assistant
- digital twin guide
- kiosk character
- brand/IP character
- object-aware explanation

### 4. Operation and Control

For B2B, the value is not only the avatar. It is also:

- knowledge base management
- persona and tone management
- forbidden topic policy
- behavior policy
- conversation logs
- latency/cost monitoring
- character profile tuning
